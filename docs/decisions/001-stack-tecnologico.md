Data Quality Platform tiene como objetivo principal servir como proyecto de portafolio para aplicar a prácticas profesionales 
de Software Engineering en septiembre de 2026. Un research informal de ofertas de trabajo para roles de práctica/junior mostró 
que React, Python y SQL aparecen de forma recurrente como tecnologías solicitadas. Adicionalmente, estas son las tecnologías que 
el autor del proyecto ya domina, lo que permite invertir el tiempo disponible (limitado, con deadline de aplicación en septiembre) 
en profundidad de arquitectura e ingeniería, en vez de en aprender un stack completamente nuevo desde cero.

Se decidió construir Data Quality Platform usando React (con TypeScript) para el frontend, Python con FastAPI para el backend,
 y PostgreSQL como base de datos, siguiendo una arquitectura hexagonal con Repository Pattern para el acceso a dato
